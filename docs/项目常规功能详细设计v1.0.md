# 2. 用户与权限管理模块详细设计

## 2.1 模块概述

用户与权限管理模块是平台的基础支撑模块，负责用户身份认证与基础信息管理，为其他业务模块提供用户身份识别与权限校验能力。

## 2.2 功能清单

| **功能编号** | **功能名称** | **功能描述** | **优先级** |
| --- | --- | --- | --- |
| F-U001 | 用户注册 | 支持新用户通过邮箱注册账号 | 高 |
| F-U002 | 用户登录 | 支持已注册用户通过用户名/邮箱和密码登录系统 | 高 |
| F-U003 | 密码找回 | 支持用户通过注册邮箱找回/重置密码 | 高 |
| F-U004 | 个人信息管理 | 支持用户查看和修改个人基本信息 | 中 |

## 2.3 功能详细设计

### 2.3.1 用户注册（F-U001）

**功能描述**：支持新用户通过邮箱注册账号，完成平台用户身份创建。

**输入参数**：

| **参数名** | **数据类型** | **必填** | **校验规则** | **说明** |
| --- | --- | --- | --- | --- |
| username | String | 是 | 长度4-50字符，仅允许字母数字下划线，唯一性校验 | 登录用户名 |
| password | String | 是 | 长度8-32字符，包含大小写字母、数字、特殊符号 | 登录密码 |
| email | String | 是 | 邮箱格式校验，唯一性校验 | 联系邮箱 |
| real_name | String | 否 | 长度≤50字符 | 用户真实姓名 |
| organization | String | 否 | 长度≤200字符 | 所属单位/院校 |

**处理流程**：

1. 接收用户注册请求，校验请求参数完整性与格式合规性
2. 校验用户名是否已存在，若存在则返回"用户名已被注册"错误
3. 校验邮箱是否已存在，若存在则返回"邮箱已被注册"错误
4. 对密码进行BCrypt加密处理
5. 创建用户记录，初始角色设为普通用户
6. 返回注册成功响应，并发送注册成功通知邮件

**输出结果**：

| **字段名** | **说明** |
| --- | --- |
| user_id | 新创建的用户唯一ID |
| username | 注册的用户名 |
| message | 注册成功提示信息 |

### 2.3.2 用户登录（F-U002）

**功能描述**：支持已注册用户通过用户名/邮箱和密码登录系统，获取访问令牌。

**输入参数**：

| **参数名** | **数据类型** | **必填** | **校验规则** | **说明** |
| --- | --- | --- | --- | --- |
| account | String | 是 | 非空 | 用户名或邮箱 |
| password | String | 是 | 非空 | 登录密码 |

**处理流程**：

1. 接收用户登录请求，校验请求参数完整性
2. 根据account查询用户记录（支持用户名或邮箱登录）
3. 若用户不存在，返回"用户不存在"错误
4. 对输入密码进行BCrypt校验，若不匹配返回"密码错误"
5. 生成JWT访问令牌（有效期24小时）和刷新令牌（有效期7天）
6. 更新用户最后登录时间
7. 返回登录成功响应，包含令牌信息

**输出结果**：

| **字段名** | **数据类型** | **说明** |
| --- | --- | --- |
| access_token | String | JWT访问令牌 |
| refresh_token | String | JWT刷新令牌 |
| token_type | String | 令牌类型（Bearer） |
| expires_in | Long | 令牌有效期（秒） |
| user_info | Object | 用户基本信息（user_id、username、email等） |

### 2.3.3 密码找回（F-U003）

**功能描述**：支持用户通过注册邮箱找回/重置密码，解决用户忘记密码的问题。

**输入参数**：

| **参数名** | **数据类型** | **必填** | **校验规则** | **说明** |
| --- | --- | --- | --- | --- |
| email | String | 是 | 邮箱格式校验 | 注册邮箱 |

**处理流程**：

1. 接收密码找回请求，校验邮箱格式
2. 查询邮箱对应的用户记录，若不存在返回"邮箱未注册"错误
3. 生成密码重置令牌（有效期30分钟），存入缓存
4. 发送密码重置邮件，包含重置链接和令牌
5. 返回"重置邮件已发送"提示

**密码重置流程**：

1. 用户点击邮件中的重置链接，前端展示密码重置页面
2. 用户输入新密码，提交重置请求（携带令牌）
3. 校验令牌有效性和有效期
4. 对新密码进行BCrypt加密，更新用户密码
5. 删除缓存中的重置令牌，返回重置成功提示

**功能描述**：支持用户查看和修改个人基本信息。

**查看个人信息**：

**处理流程**：

**输出结果**：

**修改个人信息**：

**输入参数**：

| **参数名** | **数据类型** | **必填** | **校验规则** | **说明** |
| --- | --- | --- | --- | --- |

**处理流程**：

# 3. 电解液配方管理模块详细设计

## 3.1 模块概述

电解液配方管理模块是计算引擎的核心数据模块之一，负责电解液配方的创建、编辑、查询、删除，以及分子模板库的管理，为自动化建模提供配方数据支撑。

## 3.2 功能清单

| **功能编号** | **功能名称**   | **功能描述**                                   | **优先级** |
| ------------ | -------------- | ---------------------------------------------- | ---------- |
| F-E001       | 配方创建       | 创建新的电解液配方，配置溶剂、锂盐、温度等参数 | 高         |
| F-E002       | 配方编辑       | 修改已有电解液配方的参数信息                   | 高         |
| F-E003       | 配方删除       | 删除不再需要的电解液配方                       | 中         |
| F-E004       | 配方查询       | 按条件查询电解液配方列表                       | 高         |
| F-E005       | 配方详情       | 查看电解液配方的完整参数信息                   | 高         |
| F-E006       | 配方保存为模板 | 将常用配方保存为可复用的模板                   | 中         |
| F-E007       | 配方复用       | 从已保存的模板一键创建新配方                   | 中         |

## 3.3 功能详细设计

### 3.3.1 配方创建（F-E001）

**功能描述**：创建新的电解液配方，配置溶剂种类与摩尔比、锂盐种类与浓度、温度、压力等核心参数。                                                                                                                             

**输入参数**：

| **参数名**        | **数据类型** | **必填** | **校验规则**     | **说明**          |
| ----------------- | ------------ | -------- | ---------------- | ----------------- |
| system\_name      | String       | 是       | 长度≤200字符     | 配方名称          |
| task\_description | String       | 否       | 长度≤200字符     | 任务说明          |
| solvent\_info     | JSON         | 是       | 见溶剂信息结构   | 混合溶剂信息      |
| salt\_info        | JSON         | 否       | 见锂盐信息结构   | 锂盐信息          |
| additive\_info    | JSON         | 否       | 见添加剂信息结构 | 添加剂信息        |
| temperature       | Double       | 是       | 范围200-500K     | 目标模拟温度(K)   |
| pressure          | Double       | 是       | 范围0.1-1000bar  | 目标模拟压强(bar) |
| box\_size         | JSON         | 是       | 各边长≥20Å       | 模拟盒子尺寸      |

**溶剂信息结构（solvent\_info）**：

```json
{
  "solvents": [
    {
      "molecule_id": 1,
      "molecule_name": "EC",
      "mole_fraction": 0.5
    },
    {
      "molecule_id": 2,
      "molecule_name": "DMC",
      "mole_fraction": 0.5
    }
  ]
}
```

**锂盐信息结构（salt\_info）**：

```json
{
  "cation_id": 10,
  "cation_name": "Li+",
  "anion_id": 11,
  "anion_name": "PF6-",
  "concentration_mol_per_liter": 1.0
}
```

**处理流程**：

1. 接收配方创建请求，校验请求参数完整性与格式合规性
2. 校验溶剂摩尔分数之和是否为1
3. 校验所选分子模板是否存在
4. 计算体系总原子数（基于分子数量和单分子原子数）
5. 生成唯一配方ID，创建配方记录
6. 返回配方创建成功响应

**输出结果**：

| **字段名**         | **说明**           |
| ------------------ | ------------------ |
| system\_id         | 新创建的配方唯一ID |
| system\_name       | 配方名称           |
| total\_atom\_count | 体系总原子数       |
| message            | 创建成功提示信息   |

### 3.3.2 配方编辑（F-E002）

**功能描述**：修改已有电解液配方的参数信息。

**输入参数**：同配方创建，额外包含system\_id

**处理流程**：

1. 校验配方是否存在且属于当前用户
2. 校验配方是否有关联的运行中任务，若有则禁止修改
3. 更新配方信息记录
4. 返回更新成功提示

### 3.3.3 配方删除（F-E003）

**功能描述**：删除不再需要的电解液配方。

**输入参数**：

| **参数名** | **数据类型** | **必填** | **说明** |
| ---------- | ------------ | -------- | -------- |
| system\_id | Long         | 是       | 配方ID   |

**处理流程**：

1. 校验配方是否存在且属于当前用户
2. 校验配方是否有关联的任务记录，若有则提示"存在关联任务，无法删除"
3. 删除配方记录
4. 返回删除成功提示

### 3.3.4 配方查询（F-E004）

**功能描述**：按条件查询电解液配方列表。

**输入参数**：

| **参数名** | **数据类型** | **必填** | **说明**         |
| ---------- | ------------ | -------- | ---------------- |
| keyword    | String       | 否       | 配方名称关键词   |
| page       | Integer      | 否       | 页码，默认1      |
| page\_size | Integer      | 否       | 每页条数，默认20 |

**处理流程**：

1. 从JWT令牌中解析用户ID
2. 按用户ID和关键词查询配方列表
3. 按创建时间倒序排列，分页返回结果

**输出结果**：

| **字段名** | **说明**                                                     |
| ---------- | ------------------------------------------------------------ |
| total      | 总记录数                                                     |
| page       | 当前页码                                                     |
| page\_size | 每页条数                                                     |
| list       | 配方列表（包含system\_id、system\_name、temperature、pressure、create\_time等） |

### 3.3.5 配方详情（F-E005）

**功能描述**：查看电解液配方的完整参数信息。

**输入参数**：

| **参数名** | **数据类型** | **必填** | **说明** |
| ---------- | ------------ | -------- | -------- |
| system\_id | Long         | 是       | 配方ID   |

**处理流程**：

1. 校验配方是否存在
2. 查询配方完整信息
3. 关联查询分子模板详细信息
4. 返回配方完整信息

### 3.3.6 配方保存为模板（F-E006）

**功能描述**：将常用配方保存为可复用的模板。

**输入参数**：

| **参数名** | **数据类型** | **必填** | **说明**            |
| ---------- | ------------ | -------- | ------------------- |
| system\_id | Long         | 是       | 配方ID              |
| is\_public | Boolean      | 否       | 是否公开，默认false |

**处理流程**：

1. 校验配方是否存在且属于当前用户
2. 将配方标记为模板（is\_public\_template字段更新）
3. 返回保存成功提示

### 3.3.7 配方复用（F-E007）

**功能描述**：从已保存的模板一键创建新配方。

**输入参数**：

| **参数名** | **数据类型** | **必填** | **说明**   |
| ---------- | ------------ | -------- | ---------- |
| system\_id | Long         | 是       | 模板配方ID |
| new\_name  | String       | 是       | 新配方名称 |

**处理流程**：

1. 查询模板配方信息
2. 校验模板是否为公开模板或属于当前用户
3. 复制配方信息，生成新配方记录
4. 返回新配方ID

# 4. 模拟任务全生命周期管理模块详细设计

模拟任务全生命周期管理模块是平台的核心业务模块，负责模拟任务的创建、状态监控、任务控制、任务查询等全流程管理，是连接用户操作与计算引擎的桥梁。

## 4.2 功能清单

| **功能编号** | **功能名称** | **功能描述** | **优先级** | 进展 |
| --- | --- | --- | --- | --- |
| F-J001 | 任务创建 | 创建新的模拟任务，选择配方和计算性质 | 高 | 100% |
| F-J002 | 任务状态查询 | 查询任务当前状态和执行进度 | 高 |  |
| F-J003 | 任务日志查看 | 实时查看任务执行日志 | 高 |  |
| F-J004 | 任务取消 | 取消正在运行或排队中的任务 | 高 |  |
| F-J005 | 任务删除 | 删除已完成的任务记录 | 中 |  |
| F-J006 | 任务重试 | 重新执行失败的任务 | 中 | 70% |

## 4 .3 任务状态定义

| **状态值** | **状态名称** | **状态说明** |
| --- | --- | --- |
| PENDING | 排队中 | 任务已创建，等待执行 |
| RUNNING | 运行中 | 任务正在执行模拟计算 |

## 4.4 功能详细设计

### 4.4.1 任务创建（F-J001）

**功能描述**：创建新的模拟任务，选择已有电解液配方，勾选目标计算性质，配置模拟参数。

**输入参数**：

| **参数名** | **数据类型** | **必填** | **校验规则** | **说明** |
| --- | --- | --- | --- | --- |
| job_name | String | 是 | 长度≤200字符 | 任务名称 |
| system_id | Long | 是 | 存在性校验 | 关联的电解液配方ID |
| target_properties | Array | 是 | 非空数组 | 目标计算性质列表 |
| hardware_used | String | 否 | CPU/GPU/BOTH，默认CPU | 使用硬件类型 |
| cpu_cores | Integer | 否 | 默认8 | CPU核心数 |
| simulation_params | JSON | 否 | 见模拟参数结构 | 模拟参数配置 |

**目标计算性质可选值**：
- density：密度
- conductivity：电导率
- viscosity：粘度
- dielectric_constant：介电常数
- solvation_structure：溶剂化结构

**模拟参数结构（simulation_params）**：

```json
{
  "time_step_fs": 1.0,
  "cutoff_distance_ang": 10.0,
  "equilibrium_time_ps": 700,
  "production_time_ps": 5000,
  "output_frequency_step": 1000
}
```

**处理流程**：

# 7. 数据库表详细设计

## 7.0 类包分类说明

根据项目包结构规范，数据库表对应的实体类放在`model`包，数据传输对象放在`dto`包：

| 包路径 | 用途 | 对应表 | 实体类名 |
|--------|------|--------|----------|
| `com.mdplatform.model` | 用户实体 | sys_user_table | `SysUser` |
| `com.mdplatform.model` | 模拟任务实体 | simulation_jobs_table | `SimulationJob` |
| `com.mdplatform.dto` | 系统数据传输对象 | - | `SystemDto` |
| `com.mdplatform.dto` | 模拟数据传输对象 | - | `SimulationDto` |
| `com.mdplatform.dto` | 模拟统计数据传输对象 | - | `SimulationStatsDto` |

**分类原则**：

- **model包**：使用`@Entity`注解，对应数据库表，由Repository直接操作
- **dto包**：无JPA注解，用于Service层与Controller层之间数据传递，通过`fromEntity()`方法从实体转换

## 7.1 用户表（sys_user_table）

用户表存储平台用户基本信息，是全平台用户身份识别的核心表。

| **字段名** | **数据类型** | **约束** | **字段含义** | **说明** |
| --- | --- | --- | --- | --- |
| user_id | BIGINT | PRIMARY KEY, AUTO_INCREMENT | 用户唯一ID | 全平台用户唯一标识 |
| username | VARCHAR(50) | NOT NULL, UNIQUE | 登录用户名 | 账号体系基础 |
| password | VARCHAR(100) | NOT NULL | BCrypt加密后的密码 | 账号安全 |
| real_name | VARCHAR(50) | NULL | 用户真实姓名 | 用户基本信息 |
| email | VARCHAR(100) | NOT NULL, UNIQUE | 联系邮箱 | 账号找回、通知推送 |
| organization | VARCHAR(200) | NULL | 所属单位/院校 | 用户基本信息 |
| phone | VARCHAR(20) | NULL | 联系电话 | 辅助账号管理 |
| role_id | BIGINT | NOT NULL | 关联角色ID | 权限管控 |
| last_login_time | DATETIME | NULL | 最后登录时间 | 审计需求 |
| create_time | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 账号创建时间 | 审计需求 |
| update_time | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 账号更新时间 | 审计需求 |

# 8. 业务流程详细设计

## 8.1 主业务全流程

主业务流程是用户使用平台的核心链路，覆盖从用户登录到查看计算结果的完整操作，共分为以下核心步骤：

### 步骤1：用户登录与配方选择

**操作描述**：用户完成平台登录，通过配方及计算引擎功能模块选择配方。

**详细操作**：
1. 用户输入账号密码，完成登录认证
2. 通过配方及计算引擎功能模块选择配方

### 步骤2：任务创建与参数配置

**操作描述**：用户基于配方创建模拟任务，配置计算参数。

**详细操作**：
1. 选择目标配方，点击创建任务
2. 勾选需要计算的目标性质（密度、电导率、粘度等）
3. 配置模拟参数（时间步长、模拟时长、硬件资源等）
4. 提交任务创建请求
5. 系统生成唯一任务ID，初始状态设为PENDING

### 步骤3：参数校验与任务入队

**操作描述**：后端服务校验参数合规性，准备任务执行环境。

**详细操作**：
1. 后端接收任务请求
2. 校验配方参数与模拟参数的合规性
3. 校验通过后生成唯一任务ID
4. 将任务信息存入MySQL数据库
5. 创建任务专属文件目录

### 步骤4：自动化体系构建

**操作描述**：系统自动完成分子拓扑组装与初始构型生成。

**详细操作**：
1. 解析配方参数，计算各分子数量
2. 从分子库调取单分子模板文件
3. 调用Packmol完成分子随机堆积，生成初始构型文件
4. 调用Moltemplate生成LAMMPS可识别的data结构文件

### 步骤5：输入脚本生成

**操作描述**：系统自动生成LAMMPS输入脚本。

**详细操作**：
1. 基于Jinja2模板引擎
2. 根据用户勾选的计算性质，动态生成LAMMPS输入脚本
3. 自动添加对应性质的compute、dump输出命令
4. 生成完整的输入文件包

### 步骤6：计算资源准备

**操作描述**：准备计算容器环境，注入输入文件。

**详细操作**：
1. Docker调度服务检查计算容器运行状态
2. 将生成的data文件、in文件传入容器内指定工作目录
3. 准备执行环境

### 步骤7：LAMMPS模拟执行

**操作描述**：在容器内执行LAMMPS模拟流程。

**详细操作**：
1. MD执行服务在容器内调用MPI并行命令
2. 启动LAMMPS模拟流程
3. 依次执行：能量最小化 → NVT平衡 → NPT平衡 → 生产模拟
4. 更新任务状态为RUNNING

### 步骤8：任务状态实时更新

**操作描述**：实时监控模拟进度，更新任务状态。

**详细操作**：
1. 后端实时拉取容器内的模拟日志
2. 解析任务进度
3. 更新数据库中的任务状态
4. 向前端推送实时日志与进度信息

### 步骤9：模拟结果回收

**操作描述**：模拟完成后回收输出文件。

**详细操作**：
1. 模拟执行完成后，Docker调度服务将输出文件复制到宿主机
2. 输出文件包括：log.lammps、dump轨迹文件、pressure.dat等
3. 若模拟成功，更新状态为COMPLETED
4. 若模拟失败，更新状态为FAILED，记录错误日志

### 步骤10：自动化后处理

**操作描述**：自动执行后处理分析，计算理化性质。

**详细操作**：
1. 后处理服务根据用户勾选的计算性质
2. 调用MDSuite工具，解析LAMMPS输出文件
3. 依次完成目标性质的自动化计算
4. 生成标准化的数值结果与可视化数据源

### 步骤11：结果数据存储

**操作描述**：将计算结果存入数据库和文件系统。

**详细操作**：
1. 将理化性质数值结果存入MySQL数据库
2. 将轨迹文件、图表数据源、3D结构文件存入文件系统
3. 建立与任务ID的关联关系

### 步骤12：结果可视化渲染

**操作描述**：前端渲染计算结果可视化。

**详细操作**：
1. 前端请求任务结果数据
2. 调用Plotly.js绘制二维性质曲线
3. 调用3Dmol.js加载分子轨迹文件
4. 完成3D结构渲染与轨迹动画播放

### 步骤13：结果查看与导出

**操作描述**：用户查看和导出计算结果。

**详细操作**：
1. 用户查看计算结果、数值卡片、图表与3D分子结构
2. 支持结果报告、图表、轨迹文件的下载导出
