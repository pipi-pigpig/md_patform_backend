# 项目数据库设计 v1.0

## 1. 数据库概述

### 1.1 数据库基本信息
- **数据库名称**: md_database
- **数据库类型**: MySQL (TiDB Cloud)
- **字符集**: utf8mb4
- **排序规则**: utf8mb4_unicode_ci
- **存储引擎**: InnoDB
- **连接地址**: gateway01.ap-southeast-1.prod.aws.tidbcloud.com:4000

### 1.2 数据库配置
```yaml
连接池配置:
  最大连接数: 20
  最小空闲连接数: 5
  连接超时时间: 30000ms
  空闲超时时间: 600000ms
  最大生命周期: 1800000ms

JPA配置:
  DDL模式: none
  显示SQL: false
  方言: MySQL8Dialect
  批处理大小: 20
```

### 1.3 数据表统计
本项目共包含 **10** 个数据表，分为以下几类：

| 类别 | 表名 | 说明 |
|------|------|------|
| 用户管理 | sys_user_table | 系统用户表 |
| 用户管理 | sys_role_table | 系统角色表 |
| 用户管理 | sys_operation_log_table | 系统操作日志表 |
| 电解液系统 | electrolyte_systems_table | 电解液系统配置表 |
| 模拟计算 | simulation_jobs_table | 模拟任务表 |

| 模拟计算 | calculation_result_table | 计算结果表 |
| 模拟计算 | job_execution_log_table | 任务执行日志表 |

| update_time | DATETIME | - | 否 | CURRENT_TIMESTAMP | 更新时间 |

**索引设计**:
- PRIMARY KEY: molecule_id

## 4. 索引设计策略

### 4.1 主键索引
所有表均使用自增BIGINT类型作为主键，确保高效的主键查询性能。

### 4.2 唯一索引
对需要保证唯一性的字段建立唯一索引：
- username, email (用户表)
- role_name, role_code (角色表)
- molecule_name (分子模板表)
- job_id (模拟输入表、模拟输出表)

### 4.3 外键索引
外键字段自动创建索引，优化关联查询性能。

### 4.4 业务索引
根据查询场景，建议在以下字段添加索引：
- simulation_jobs_table.status (按状态查询任务)
- simulation_jobs_table.create_time (按时间查询任务)
- calculation_result_table.property_name (按属性名称查询结果)

## 5. 数据完整性约束

### 5.1 非空约束
关键字段设置为NOT NULL，确保数据完整性：
- 用户名、密码
- 系统名称、溶剂信息
- 任务名称、软件名称、状态
- 属性名称、属性值、单位

### 5.2 默认值约束

- status: 'PENDING' (任务状态)
- hardware_used: 'CPU' (硬件类型)
- cpu_cores: '8' (CPU核心数)
- time_step_fs: 1.0 (时间步长)
- is_public_template: 0 (是否公开)

使用JSON类型存储复杂结构数据：
- solvent_info, salt_info, additive_info (电解液系统)
- target_properties, result_summary (模拟任务)
- minimization_params, equilibrium_params, production_params (模拟输入)
