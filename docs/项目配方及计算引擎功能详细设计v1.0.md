# 3. 计算引擎集成与调度模块详细设计

## 3.1 模块概述

计算引擎集成与调度模块是平台的算力核心，负责LAMMPS分子动力学计算引擎的容器化封装、任务调度执行及容错处理。该模块基于Docker容器化技术实现，与宿主机环境解耦，确保计算环境的一致性与可移植性。

## 3.2 功能清单

| 功能编号 |   功能名称   |                           功能描述                           | 优先级 |
| :------: | :----------: | :----------------------------------------------------------: | :----: |
|  F-C001  | 任务调度执行 | 自动化执行容器内 LAMMPS 命令，注入输入文件、实时采集执行日志 |   高   |
|  F-C002  | 计算结果回收 |         从容器导出模拟输出文件，校验完整性并写入存储         |   高   |
|  F-C003  | 任务超时监控 |     定时检测运行中任务，超时时自动终止容器并标记任务状态     |   高   |
|  F-C004  | 错误日志记录 |            分类记录任务执行错误，生成故障排查建议            |   高   |

## 3.3 功能详细设计

#### 3.3.1 任务调度执行（F-C001）

**功能描述**：启动指定计算容器，注入 LAMMPS 输入文件，执行 MPI 并行模拟命令，实时采集执行日志并解析模拟进度，自动更新任务状态。

**输入参数**：

|    参数名    | 数据类型 | 必填 |                       校验规则                        |         说明          |
| :----------: | :------: | :--: | :---------------------------------------------------: | :-------------------: |
|    job_id    |   Long   |  是  | 存在于`simulation_jobs_table`，状态为`PENDING/PAUSED` |      任务唯一 ID      |
| container_id |  String  |  是  | 匹配 Docker 容器 ID 格式，容器状态为`Created/Exited`  |      计算容器 ID      |
|  mpi_procs   | Integer  |  是  |               1 ≤ mpi_procs ≤ cpu_cores               |    MPI 并行进程数     |
|  input_file  |  String  |  是  |         容器内路径：`/workspace/inputs/*.in`          | LAMMPS 输入脚本文件名 |
|   log_file   |  String  |  是  |        容器内路径：`/workspace/results/*.log`         | LAMMPS 日志输出文件名 |

**处理流程**：

1. 校验任务状态为`PENDING/PAUSED`，容器状态为`Created/Exited`；

2. 启动容器（`docker start {container_id}`），等待容器进入`Running`状态；

3. 校验容器内`/workspace/inputs/{input_file}`文件存在且大小 > 0；

4. 构建 LAMMPS 执行命令：

	bash

	```
	mpirun -np {mpi_procs} --allow-run-as-root lmp_gpu -in {input_file} -log {log_file}
	```

5. 在容器内异步执行上述命令，同时启动两个线程：

	- **日志采集线程**：实时读取`stdout/stderr`流，每 100 行写入`job_execution_log_table`；

	- 进度解析线程

		：每 60 秒读取一次log_file，通过正则表达式匹配Step Temp E_pair TotEng Press行，解析当前模拟步数，计算进度百分比：

		```
		进度 = (当前步数 / 总步数) × 100%
		```

6. 等待命令执行完成，获取退出码：

	- 退出码 = 0：模拟成功，更新任务状态为`COMPLETED`；
	- 退出码≠0：模拟失败，更新任务状态为`FAILED`，提取最后 100 行错误日志；

7. 记录任务执行时长（`end_time - start_time`）到`simulation_jobs_table`；

8. 返回任务执行结果。

**输出结果**：

|       字段名       |              说明               |
| :----------------: | :-----------------------------: |
|       job_id       |             任务 ID             |
|     exit_code      |     命令退出码（0 为成功）      |
| execution_duration |         执行时长（秒）          |
|  progress_percent  |         最终进度百分比          |
|       status       | 任务状态（`COMPLETED/FAILED`）  |
|     error_log      | 失败时的错误日志（最后 100 行） |

#### 3.3.2 计算结果回收（F-C002）

**功能描述**：模拟任务完成后，从容器导出所有输出文件到宿主机，校验文件完整性，更新文件路径到数据库。

**输入参数**：

|    参数名    | 数据类型 | 必填 |                     校验规则                     |          说明          |
| :----------: | :------: | :--: | :----------------------------------------------: | :--------------------: |
|    job_id    |   Long   |  是  | 存在于`simulation_jobs_table`，状态为`COMPLETED` |      任务唯一 ID       |
| container_id |  String  |  是  |   匹配 Docker 容器 ID 格式，容器状态为`Exited`   |      计算容器 ID       |
| output_path  |  String  |  是  |          宿主机路径，当前用户有读写权限          | 宿主机任务输出文件目录 |

**处理流程**：

1. 校验任务状态为`COMPLETED`，容器状态为`Exited`；
2. 获取容器内`/workspace/results/`目录下的所有文件列表；
3. 遍历文件列表，通过 Docker API 将每个文件复制到宿主机`output_path`目录；
4. 校验所有必需输出文件存在且大小 > 0：
	- 必需文件：`log.lammps`、`dump.trajectory.lammpstrj`、`equilibrated.data`、`final.data`；
	- 可选文件：`pressure.dat`、`dipole.dat`、`thermo.out`；
5. 计算每个文件的 MD5 校验和，写入`simulation_raw_output_table`；
6. 将所有文件的宿主机路径、大小、校验和更新到数据库；
7. 若所有文件校验通过，返回回收成功；若有文件缺失或损坏，标记任务为`PARTIAL_SUCCESS`，记录缺失文件列表。

**输出结果**

|      字段名      |            说明            |
| :--------------: | :------------------------: |
|     success      | 回收是否成功（true/false） |
|   total_files    |       回收的文件总数       |
|  missing_files   |   缺失的文件列表（数组）   |
| corrupted_files  |   损坏的文件列表（数组）   |
| output_directory |     宿主机输出文件目录     |

#### 3.3.3 任务超时监控（F-C003）

**功能描述**：定时扫描运行中任务，分阶段计算已执行时长，超过预设超时阈值时自动终止容器，标记任务为`TIMEOUT`并记录原因。

**输入参数**：无（定时任务，每 60 秒自动执行一次）

**处理流程**：

1. 从`simulation_jobs_table`筛选所有状态为`RUNNING`的任务；

2. 对每个任务，从`job_execution_log_table`解析当前模拟阶段（能量最小化 / NVT 平衡 / NPT 平衡 / 生产模拟）；

3. 根据当前阶段获取对应超时阈值（可配置）：

	|  模拟阶段  |     默认超时阈值     |
	| :--------: | :------------------: |
	| 能量最小化 |  3600 秒（1 小时）   |
	|  NVT 平衡  |  7200 秒（2 小时）   |
	|  NPT 平衡  |  14400 秒（4 小时）  |
	|  生产模拟  | 172800 秒（48 小时） |
	| 任务总时长 | 259200 秒（72 小时） |

4. 计算当前阶段已执行时长：`当前时间 - 阶段开始时间`；

5. 若已执行时长 > 对应阶段超时阈值，或总执行时长 > 总超时阈值：

	- 调用`docker stop {container_id}`强制终止容器；
	- 更新任务状态为`TIMEOUT`，写入错误信息：`任务在{阶段}阶段超时，已执行{时长}秒`；
	- 记录超时日志到`job_execution_log_table`；
	- 发送超时告警通知（站内信 / 邮件）；

6. 继续扫描下一个任务。

**输出结果**：无（更新数据库状态 + 日志 + 告警）

#### 3.3.4 错误日志记录（F-C004）

**功能描述**：分类记录任务执行过程中的所有日志信息，支持按级别、时间、任务 ID 检索，为故障排查提供依据。

**输入参数**：

|   参数名    | 数据类型 | 必填 |                校验规则                 |               说明                |
| :---------: | :------: | :--: | :-------------------------------------: | :-------------------------------: |
|   job_id    |   Long   |  是  |      存在于`simulation_jobs_table`      |            任务唯一 ID            |
|  log_level  |  String  |  是  |     枚举值：`DEBUG/INFO/WARN/ERROR`     |             日志级别              |
| log_content |  String  |  是  |          非空，长度≤65535 字符          |           日志详细内容            |
| error_type  |  String  |  否  | 如 "输入文件错误 / 内存溢出 / MPI 错误" |   错误类型（仅 ERROR 级别需要）   |
| suggestion  |  String  |  否  |              长度≤500 字符              | 故障排查建议（仅 ERROR 级别需要） |

**处理流程**：

1. 校验`job_id`存在；
2. 校验`log_level`为合法枚举值；
3. 截取`log_content`到最大长度（65535 字符）；
4. 生成当前时间戳；
5. 将日志记录插入`job_execution_log_table`；
6. 若为`ERROR`级别日志，同时更新任务的`error_message`字段。

**输出结果**：

| 字段名  |            说明            |
| :-----: | :------------------------: |
| log_id  |     生成的日志唯一 ID      |
| success | 记录是否成功（true/false） |

### 3.4.3 核心类设计

**DockerSchedulerService类**

```java
@Service
public class DockerSchedulerService {
    
    private DockerClient dockerClient;
    
    public String createComputeContainer(JobConfig config) {
        CreateContainerResponse response = dockerClient.createContainerCmd("md-engine:latest")
            .withName("job-" + config.getJobId())
            .withHostConfig(HostConfig.newHostConfig()
                .withBinds(
                    new Bind(config.getInputPath(), new Volume("/workspace/inputs")),
                    new Bind(config.getOutputPath(), new Volume("/workspace/results"))
                )
                .withCpuCount(config.getCpuCores())
                .withMemory(config.getMemoryLimit())
            )
            .withEnv(
                "OMP_NUM_THREADS=" + config.getCpuCores(),
                "CUDA_VISIBLE_DEVICES=" + config.getGpuIds()
            )
            .exec();
        return response.getId();
    }
    
    public ExecutionResult executeLammps(String containerId, LammpsConfig config) {
        String[] command = buildLammpsCommand(config);
        ExecCreateCmdResponse exec = dockerClient.execCreateCmd(containerId)
            .withCmd(command)
            .withAttachStdout(true)
            .withAttachStderr(true)
            .exec();
        return executeAndCollectOutput(exec);
    }
    
    private String[] buildLammpsCommand(LammpsConfig config) {
        List<String> cmd = new ArrayList<>();
        cmd.add("mpirun");
        cmd.add("-np");
        cmd.add(String.valueOf(config.getMpiProcs()));
        cmd.add("lmp_gpu");
        cmd.add("-in");
        cmd.add(config.getInputFile());
        return cmd.toArray(new String[0]);
    }
}
```

**MDExecutionService类**

```java
@Service
public class MDExecutionService {
    
    @Autowired
    private DockerSchedulerService dockerService;
    
    @Autowired
    private JobStatusService statusService;
    
    public void executeSimulation(Job job) {
        String containerId = null;
        try {
            containerId = dockerService.createComputeContainer(job.getConfig());
            dockerService.startContainer(containerId);
            
            statusService.updateStatus(job.getJobId(), JobStatus.RUNNING);
            
            injectInputFiles(containerId, job);
            
            ExecutionResult result = dockerService.executeLammps(containerId, job.getLammpsConfig());
            
            collectOutputFiles(containerId, job);
            
            statusService.updateStatus(job.getJobId(), 
                result.isSuccess() ? JobStatus.COMPLETED : JobStatus.FAILED);
                
        } catch (Exception e) {
            statusService.updateStatus(job.getJobId(), JobStatus.FAILED);
            logError(job.getJobId(), e);
        } finally {
            if (containerId != null) {
                dockerService.removeContainer(containerId);
            }
        }
    }
}
```

## 3.5 执行容错

### 3.5.1 设计目标

支持任务超时自动终止、执行失败自动重试，记录详细错误日志便于问题排查，确保计算任务的可靠执行。

### 3.5.2 超时控制设计

**超时配置参数**

| **参数名称**              | **默认值** | **说明**          |
| --------------------- | ------- | --------------- |
| minimization\_timeout | 3600秒   | 能量最小化阶段超时时间     |
| equilibrium\_timeout  | 7200秒   | 平衡模拟阶段超时时间      |
| production\_timeout   | 172800秒 | 生产模拟阶段超时时间      |
| total\_job\_timeout   | 259200秒 | 任务总执行超时时间（72小时） |

**超时检测实现**

```java
@Component
public class JobTimeoutMonitor {
    
    @Scheduled(fixedRate = 60000)
    public void checkTimeout() {
        List<Job> runningJobs = jobRepository.findByStatus(JobStatus.RUNNING);
        
        for (Job job : runningJobs) {
            long elapsedSeconds = calculateElapsedSeconds(job.getStartTime());
            long timeoutSeconds = getTimeoutForCurrentPhase(job);
            
            if (elapsedSeconds > timeoutSeconds) {
                handleTimeout(job);
            }
        }
    }
    
    private void handleTimeout(Job job) {
        dockerService.stopContainer(job.getContainerId());
        job.setStatus(JobStatus.TIMEOUT);
        job.setErrorMessage("任务执行超时，已自动终止");
        jobRepository.save(job);
        
        logService.log(job.getJobId(), LogLevel.WARN,
```

#### 4.3.1 分子数量计算（F-M001）

**功能描述**：基于电解液配方参数，结合物理化学公式精确计算各溶剂、锂盐、添加剂的分子 / 离子数量，为后续构型生成提供准确的输入。

**输入参数**：

|    参数名     | 数据类型 | 必填 |                   校验规则                    |                说明                |
| :-----------: | :------: | :--: | :-------------------------------------------: | :--------------------------------: |
| solvent_info  |   JSON   |  是  | 符合原文档 2.3.1 溶剂信息结构，摩尔比之和 = 1 |       混合溶剂信息（摩尔比）       |
|   salt_info   |   JSON   |  否  |         符合原文档 2.3.1 锂盐信息结构         |       锂盐信息（浓度 mol/L）       |
| additive_info |   JSON   |  否  |        同溶剂信息结构，摩尔比之和≤0.1         | 添加剂信息（摩尔比，相对于总溶剂） |
|   box_size    |   JSON   |  是  |   各边长≥20Å，格式`{"x":xx,"y":xx,"z":xx}`    |         模拟盒子尺寸（Å）          |
|  temperature  |  Double  |  是  |           200 ≤ temperature ≤ 500K            |           模拟温度（K）            |

**处理流程**：

1. 校验输入参数格式，确保`solvent_info`中所有溶剂的摩尔比之和 = 1.0（误差≤0.001）；

2. 计算盒子体积：

	plaintext

	```
	V_Å³ = box_x × box_y × box_z
	V_cm³ = V_Å³ × 10⁻²⁴
	V_L = V_Å³ × 10⁻³⁰
	```

3. 根据温度查表获取电解液的目标密度（ρ_target，单位 g/cm³）：

	- 298K 时，常规碳酸酯电解液密度≈1.2 g/cm³；
	- 温度每升高 10K，密度降低约 0.01 g/cm³；

4. 计算体系总质量：`total_mass_g = ρ_target × V_cm³`；

5. 计算各组分的摩尔质量（从`molecule_template_table`查询）；

6. 计算溶剂总摩尔数：

	plaintext

	```
	total_solvent_mol = total_mass_g / (平均溶剂摩尔质量 + 盐浓度×盐摩尔质量)
	```

7. 计算各溶剂分子数：

	plaintext

	```
	N_solvent_i = total_solvent_mol × mole_fraction_i × 6.022×10²³
	```

	结果取整，确保所有溶剂分子数之和 = total_solvent_mol×6.022×10²³；

8. 计算锂盐分子数：

	plaintext

	```
	N_salt = salt_concentration_mol/L × V_L × 6.022×10²³
	```

	拆分为阳离子数 = N_salt，阴离子数 = N_salt；

9. 计算添加剂分子数（同溶剂逻辑）；

10. 汇总总原子数：`total_atoms = Σ(N_i × atom_count_i)`；

11. 校验体系电中性：`总正电荷 = 总负电荷`（误差≤0.001）；

12. 返回计算结果。

**输出结果**：

|         字段名          |                     说明                     |
| :---------------------: | :------------------------------------------: |
|     solvent_counts      | Map<String, Integer>，键：分子名，值：分子数 |
|    salt_cation_count    |             Integer，阳离子数量              |
|    salt_anion_count     |             Integer，阴离子数量              |
|     additive_counts     |      Map<String, Integer>，添加剂分子数      |
|    total_atom_count     |            Integer，体系总原子数             |
| is_electrically_neutral |           Boolean，体系是否电中性            |
|    estimated_density    |       Double，估算的体系密度（g/cm³）        |

## 4.4 体系自动构建

### 4.4.1 设计目标

基于用户输入的配方参数，自动计算各分子数量，调用Moltemplate完成分子拓扑与力场参数组装，调用Packmol完成模拟盒子内分子随机堆积，生成无原子重叠的初始构型。

### 4.4.2 分子数量计算算法

**输入参数**

| **参数**         | **数据类型** | **说明**              |
| -------------- | -------- | ------------------- |
| solvent\_info  | JSON     | 混合溶剂信息，包含种类、摩尔比     |
| salt\_info     | JSON     | 锂盐信息，包含种类、浓度（mol/L） |
| additive\_info | JSON     | 添加剂信息               |
| box\_size      | JSON     | 模拟盒子尺寸（Å）           |
| temperature    | double   | 目标温度（K）             |

**计算公式**

```
盒子体积 V = box_x × box_y × box_z (Å³)

溶剂总摩尔数 N_solvent = 根据目标密度和盒子体积计算

各溶剂分子数 = N_solvent × 溶剂摩尔比 × 阿伏伽德罗常数

锂盐分子数 = 盐浓度(mol/L) × V(Å³) × 10⁻³⁰ × 阿伏伽德罗常数

阳离子数 = 锂盐分子数
阴离子数 = 锂盐分子数
```

**实现代码**

```java
@Service
public class MoleculeCountCalculator {
    
    public MoleculeCountResult calculate(ElectrolyteSystem system) {
        double boxVolume = calculateBoxVolume(system.getBoxSize());
        
        Map<String, Integer> solventCounts = calculateSolventCounts(
            system.getSolventInfo(), boxVolume, system.getTemperature());
        
        Map<String, Integer> saltCounts = calculateSaltCounts(
            system.getSaltInfo(), boxVolume);
        
        Map<String, Integer> additiveCounts = calculateAdditiveCounts(
            system.getAdditiveInfo(), boxVolume);
        
        int totalAtoms = calculateTotalAtoms(
            solventCounts, saltCounts, additiveCounts, system.getMoleculeTemplates());
        
        return new MoleculeCountResult(
            solventCounts, saltCounts, additiveCounts, totalAtoms);
    }
    
    private Map<String, Integer> calculateSolventCounts(SolventInfo info, double volume, double temp) {
        Map<String, Integer> counts = new HashMap<>();
        double totalMoles = estimateSolventMoles(volume, temp);
        
        for (SolventComponent comp : info.getComponents()) {
            int moleculeCount = (int) Math.round(
                totalMoles * comp.getMoleFraction() * AVOGADRO);
            counts.put(comp.getMoleculeName(), moleculeCount);
        }
        return counts;
    }
}
```

```java
@Service
public class ParameterValidationService {
    
    public ValidationResult validate(JobConfig config) {
        ValidationResult result = new ValidationResult();
        
        validateTimeStep(config, result);
        validateCutoff(config, result);
        validateSimulationTime(config, result);
        validateBoxSize(config, result);
        validateTemperature(config, result);
        
        return result;
    }
    
    private void validateTimeStep(JobConfig config, ValidationResult result) {
        double timestep = config.getTimeStep();
        
        if (timestep > 2.0) {
            result.addError("time_step", "时间步长" + timestep + "fs过大，可能导致模拟不稳定，建议≤2.0fs");
        }
        
        if (timestep != 1.0 && timestep <= 2.0) {
            result.addWarning("time_step", "推荐使用1.0fs时间步长以获得最佳稳定性");
        }
        
        if (config.getTemperature() > 400 && timestep > 0.5) {
            result.addWarning("time_step", "高温体系(" + config.getTemperature() + "K)建议减小时间步长至0.5fs以下");
        }
    }
    
    private void validateCutoff(JobConfig config, ValidationResult result) {
        double cutoff = config.getCutoffDistance();
        double minBoxSize = getMinBoxSize(config.getBoxSize());
        
        if (cutoff < 8.0) {
            result.addWarning("cutoff", "截断半径" + cutoff + "Å较小，可能影响计算精度，建议≥8.0Å");
        }
        
        if (cutoff >= minBoxSize / 2) {
            result.addError("cutoff", "截断半径不能超过盒子最小尺寸的一半(" + minBoxSize/2 + "Å)");
        }
    }
    
    private void validateSimulationTime(JobConfig config, ValidationResult result) {
        long productionSteps = config.getProductionSteps();
        double timestep = config.getTimeStep();
        double productionTimePs = productionSteps * timestep / 1000.0;
        
        if (productionTimePs < 1000) {
            result.addWarning("production_time", "生产模拟时间" + productionTimePs + "ps较短，统计误差可能较大");
        }
        
        if (config.getTargetProperties().contains("conductivity") && productionTimePs < 5000) {
            result.addWarning("production_time", "电导率计算建议生产模拟时间≥5000ps以获得收敛结果");
        }
    }
}
```

# 5. 后处理与性质计算模块详细设计

## 5.1 模块概述

后处理与性质计算模块负责对LAMMPS模拟输出的原始数据进行解析与处理，自动计算目标理化性质，生成标准化的计算结果。该模块集成MDSuite专业后处理工具，实现轨迹文件解析与性质计算的自动化。

## 5.2 功能清单

| 功能编号 |   功能名称   |                       功能描述                        | 优先级 |
| :------: | :----------: | :---------------------------------------------------: | :----: |
|  F-P001  | 轨迹文件解析 | 解析 LAMMPS dump 文件，提取原子坐标、速度、受力等数据 |   高   |
|  F-P002  | 理化性质计算 | 计算密度、扩散系数、径向分布函数（RDF）等关键理化性质 |   高   |

## 5.3 功能详细设计

#### 5.3.1 轨迹文件解析（F-P001）

**功能描述**：解析 LAMMPS 输出的 dump 轨迹文件和 log 日志文件，提取原子坐标、速度、受力、热力学数据等，为后续性质计算提供标准化的数据输入。

**输入参数**：

|     参数名     | 数据类型 | 必填 |           校验规则            |     说明     |
| :------------: | :------: | :--: | :---------------------------: | :----------: |
|     job_id     |   Long   |  是  | 存在于`simulation_jobs_table` | 任务唯一 ID  |
| dump_file_path |  String  |  是  | 文件存在且为 LAMMPS dump 格式 | 轨迹文件路径 |
| log_file_path  |  String  |  是  | 文件存在且为 log.lammps 格式  | 日志文件路径 |

**处理流程**：

1. 使用 MDAnalysis 加载轨迹文件和拓扑文件（`final.data`）；
2. 解析热力学数据（从 log 文件）：
	- 提取时间步、温度、压力、能量、密度等数据；
	- 生成时间序列数组；
3. 解析轨迹数据（从 dump 文件）：
	- 提取每帧的原子坐标、速度、受力、电荷；
	- 执行坐标解包裹（消除周期性边界影响）；
4. 将解析后的数据保存为 HDF5 格式（高效存储）；
5. 返回解析结果的元数据。

**输出结果**：

|       字段名       |         说明         |
| :----------------: | :------------------: |
|   hdf5_file_path   | 解析后的数据文件路径 |
|    total_frames    |      轨迹总帧数      |
|    timestep_fs     |    时间步长（fs）    |
| simulation_time_ns |   总模拟时间（ns）   |
|     atom_count     |       原子总数       |

#### 5.3.2 理化性质计算（F-P002）

**功能描述**：基于解析后的轨迹数据，调用 MDAnalysis 计算电解液的关键理化性质，所有计算方法均符合 CSTM 行业标准。

**输入参数**：

|     参数名     | 数据类型 | 必填 |           校验规则            |           说明           |
| :------------: | :------: | :--: | :---------------------------: | :----------------------: |
|     job_id     |   Long   |  是  | 存在于`simulation_jobs_table` |       任务唯一 ID        |
| hdf5_file_path |  String  |  是  |    文件存在且为 HDF5 格式     | 解析后的轨迹数据文件路径 |
|   properties   |  Array   |  是  |    包含需要计算的性质列表     |       目标计算性质       |
|  temperature   |  Double  |  是  |        与模拟温度一致         |      模拟温度（K）       |

**处理流程**：

1. 加载 HDF5 格式的轨迹数据；
2. 跳过前 20% 的平衡阶段数据，仅使用后 80% 的生产模拟数据进行计算；
3. 分性质计算：
	- **密度**：计算所有帧的密度平均值和标准差；
	- **扩散系数**：计算均方位移（MSD），通过 Einstein 关系拟合扩散系数；
	- **径向分布函数（RDF）**：计算指定原子对（如 Li+-O）的 g (r)，积分得到配位数；
	- **粘度**：通过 Green-Kubo 方法积分应力张量自相关函数；
	- **电导率**：通过 Green-Kubo 方法积分电流自相关函数；
	- **介电常数**：通过偶极矩涨落法计算；
4. 校验每个性质的收敛性：
	- 密度：标准差 / 平均值 < 5%；
	- 扩散系数：MSD 曲线线性相关系数 > 0.99；
	- RDF：曲线平滑，无异常峰；
5. 将计算结果写入`calculation_result_table`；
6. 返回所有性质的计算结果。

**输出结果**：

|       字段名        |          说明          |
| :-----------------: | :--------------------: |
|       density       |     密度（g/cm³）      |
|     density_std     |       密度标准差       |
|    li_diffusion     | Li + 扩散系数（m²/s）  |
|   anion_diffusion   | 阴离子扩散系数（m²/s） |
|      rdf_data       |  JSON 格式的 RDF 数据  |
| coordination_number | Li + 第一配位层配位数  |
|      viscosity      |     粘度（mPa・s）     |
|    conductivity     |     电导率（S/m）      |
| dielectric_constant |        介电常数        |
| convergence_status  |    各性质的收敛状态    |

```java
@Service
public class LogFileParser {
    
    public LogData parseLogFile(String filePath) {
        LogData data = new LogData();
        List<String> lines = Files.readAllLines(Paths.get(filePath));
        
        boolean inDataSection = false;
        for (String line : lines) {
            if (line.contains("Step Temp E_pair E_mol TotEng Press Volume Density")) {
                inDataSection = true;
                continue;
            }
            
            if (inDataSection && line.trim().isEmpty()) {
                inDataSection = false;
                continue;
            }
            
            if (inDataSection) {
                ThermoEntry entry = parseThermoLine(line);
                data.addThermoEntry(entry);
            }
        }
        
        return data;
    }
    
    private ThermoEntry parseThermoLine(String line) {
        String[] parts = line.trim().split("\\s+");
        return ThermoEntry.builder()
            .step(Long.parseLong(parts[0]))
            .temperature(Double.parseDouble(parts[1]))
            .potentialEnergy(Double.parseDouble(parts[2]))
            .totalEnergy(Double.parseDouble(parts[4]))
            .pressure(Double.parseDouble(parts[5]))
            .volume(Double.parseDouble(parts[6]))
            .density(Double.parseDouble(parts[7]))
            .build();
    }
}
```

```java
@Service
public class DensityCalculator {
    
    public DensityResult calculate(LogData logData, EquilibriumConfig config) {
        List<ThermoEntry> entries = logData.getThermoEntries();
        
        int startIndex = (int) (entries.size() * config.getEquilibrationFraction());
        List<Double> densityValues = entries.subList(startIndex, entries.size())
            .stream()
            .map(ThermoEntry::getDensity)
            .collect(Collectors.toList());
        
        double mean = StatisticsUtil.mean(densityValues);
        double std = StatisticsUtil.standardDeviation(densityValues);
        double error = std / Math.sqrt(densityValues.size());
        
        return DensityResult.builder()
            .value(mean)
            .unit("g/cm³")
            .standardDeviation(std)
            .standardError(error)
            .sampleSize(densityValues.size())
            .calculationMethod("时间序列平均")
            .convergenceStatus(std / mean < 0.05 ? "converged" : "unconverged")
            .build();
    }
}
```

```java
@Service
public class ViscosityCalculator {
    
    public ViscosityResult calculate(String pressureFilePath, ViscosityConfig config) {
        List<PressureTensor> tensors = parsePressureFile(pressureFilePath);
        
        double[] autoCorrelation = calculateStressAutocorrelation(tensors, config.getMaxLag());
        
        double viscosity = integrateGreenKubo(autoCorrelation, config.getTimestep(), config.getTemperature());
        
        double convergenceError = estimateConvergenceError(autoCorrelation);
        
        return ViscosityResult.builder()
            .value(viscosity)
            .unit("mPa·s")
            .calculationMethod("Green-Kubo")
            .correlationFunction(autoCorrelation)
            .convergenceError(convergenceError)
            .convergenceStatus(convergenceError / viscosity < 0.1 ? "converged" : "unconverged")
            .build();
    }
    
    private double[] calculateStressAutocorrelation(List<PressureTensor> tensors, int maxLag) {
        int n = tensors.size();
        double[] acf = new double[maxLag];
        
        double[] stress = tensors.stream()
            .mapToDouble(t -> t.getOffDiagonalNorm())
            .toArray();
        
        double mean = Arrays.stream(stress).average().orElse(0);
        double[] fluctuation = Arrays.stream(stress).map(s -> s - mean).toArray();
        
        for (int lag = 0; lag < maxLag; lag++) {
            double sum = 0;
            for (int i = 0; i < n - lag; i++) {
                sum += fluctuation[i] * fluctuation[i + lag];
            }
            acf[lag] = sum / (n - lag);
        }
        
        return acf;
    }
    
    private double integrateGreenKubo(double[] acf, double timestep, double temperature) {
        double kb = 1.380649e-23;
        double volume = getSystemVolume();
        
        double integral = 0;
        for (int i = 0; i < acf.length - 1; i++) {
            integral += (acf[i] + acf[i + 1]) * timestep / 2;
        }
        
        return integral * volume / (3 * kb * temperature) * 1e-3;
    }
}
```

### 6.2.2 步骤详细说明

**步骤1：接收配方参数**

| **输入参数**       | **数据类型** | **说明**                                                                             |
| -------------- | -------- | ---------------------------------------------------------------------------------- |
| solvent\_info  | JSON     | 混合溶剂信息，格式：`[{"name":"EC","mole_fraction":0.5},{"name":"DMC","mole_fraction":0.5}]` |
| salt\_info     | JSON     | 锂盐信息，格式：`{"cation":"Li","anion":"PF6","concentration":1.0}`                        |
| additive\_info | JSON     | 添加剂信息（可选）                                                                          |
| box\_size      | JSON     | 模拟盒子尺寸，格式：`{"x":40.0,"y":40.0,"z":40.0}`                                           |
| temperature    | double   | 目标温度（K）                                                                            |

**步骤2：计算分子数量**

计算逻辑：

1. 计算盒子体积：V = box\_x × box\_y × box\_z (Å³)
2. 根据电解液典型密度（约1.2 g/cm³）估算溶剂总摩尔数
3. 根据摩尔比计算各溶剂分子数
4. 根据盐浓度计算锂盐分子数
5. 验证体系电中性：总正电荷 = 总负电荷

**步骤3：调取分子模板**

从分子模板库（molecule\_template\_table）调取以下文件：

- 单分子结构文件：`{molecule_name}.pdb` - 包含单个分子的3D坐标
- Moltemplate模板文件：`{molecule_name}.lt` - 定义分子拓扑、力场参数

**步骤4：Packmol堆积**

Packmol输入脚本示例：

```
tolerance 2.0
filetype pdb
output packed_system.pdb

structure EC.pdb
  number 200
  inside box 0 0 0 40.0 40.0 40.0
end structure

structure DMC.pdb
  number 200
  inside box 0 0 0 40.0 40.0 40.0
end structure

structure Li.pdb
  number 50
  inside box 0 0 0 40.0 40.0 40.0
end structure

structure PF6.pdb
  number 50
  inside box 0 0 0 40.0 40.0 40.0
end structure
```

**步骤5：生成system.lt**

system.lt文件示例：

```
import "EC.lt"
import "DMC.lt"
import "Li.lt"
import "PF6.lt"

write_once("Data Boundary") {
  0.0 40.0 xlo xhi
  0.0 40.0 ylo yhi
  0.0 40.0 zlo zhi
}

create_atoms EC random 200 12345 packed_system.pdb EC
create_atoms DMC random 200 12346 packed_system.pdb DMC
create_atoms Li random 50 12347 packed_system.pdb Li
create_atoms PF6 random 50 12348 packed_system.pdb PF6
```

**步骤6：执行Moltemplate**

执行命令：

```bash
moltemplate.sh system.lt -pdb packed_system.pdb
```

输出文件：

- `system.data` - 包含原子坐标、拓扑、力场参数的完整结构文件
- `system.in.init` - 模拟初始化设置
- `system.in.settings` - 力场补充设置

**步骤7：文件整理输出**

将生成的文件整理到任务输入目录：

```
job_{job_id}/inputs/
├── system.lt
├── system.data
├── system.in.init
├── system.in.settings
├── packmol.inp
└── packed_system.pdb
```

## 8.3 计算方法说明

### 8.3.1 Green-Kubo方法

**电导率计算公式**

$$\sigma = \frac{1}{3Vk\_BT} \int\_0^\infty \langle \mathbf{J}(0) \cdot \mathbf{J}(t) \rangle dt$$

其中：

- $\sigma$：电导率
- $V$：体系体积
- $k\_B$：玻尔兹曼常数
- $T$：温度
- $\mathbf{J}$：电流密度

**粘度计算公式**

$$\eta = \frac{V}{3k\_BT} \int\_0^\infty \langle P\_{\alpha\beta}(0) \cdot P\_{\alpha\beta}(t) \rangle dt$$

其中：

- $\eta$：粘度
- $P\_{\alpha\beta}$：非对角压力张量分量

### 8.3.2 径向分布函数

**RDF定义**

$$g\_{AB}(r) = \frac{V}{N\_A N\_B} \left\langle \sum\_{i \in A} \sum\_{j \in B} \delta(r - r\_{ij}) \right\rangle$$

**配位数计算**

$$n\_{AB} = 4\pi \rho\_B \int\_0^{r\_c} r^2 g\_{AB}(r) dr$$

# 11. 文件系统结构详细设计

## 11.1 设计原则

1. **用户隔离**：不同用户的文件存储在独立目录，禁止跨用户访问
2. **任务隔离**：每个任务拥有独立的工作目录，文件互不干扰
3. **结构清晰**：按功能模块划分目录，便于文件管理与检索
4. **与数据库绑定**：文件路径与数据库记录一一对应，支持快速定位

## 11.5 文件存储服务接口

```java
@Service
public class FileStorageService {
    
    public String getJobRootPath(Long userId, Long jobId) {
        return String.format("md_platform_data/user_%d/jobs/job_%d/", userId, jobId);
    }
    
    public String getInputPath(Long userId, Long jobId) {
        return getJobRootPath(userId, jobId) + "inputs/";
    }
    
    public String getOutputPath(Long userId, Long jobId) {
        return getJobRootPath(userId, jobId) + "raw_outputs/";
    }
    
    public String getPostProcessingPath(Long userId, Long jobId) {
        return getJobRootPath(userId, jobId) + "post_processing/";
    }
    
    public String getVisualizationPath(Long userId, Long jobId) {
        return getJobRootPath(userId, jobId) + "visualization/";
    }
    
    public void createJobDirectory(Long userId, Long jobId) {
        String rootPath = getJobRootPath(userId, jobId);
        Files.createDirectories(Paths.get(rootPath + "inputs"));
        Files.createDirectories(Paths.get(rootPath + "raw_outputs"));
        Files.createDirectories(Paths.get(rootPath + "post_processing"));
        Files.createDirectories(Paths.get(rootPath + "visualization/charts"));
        Files.createDirectories(Paths.get(rootPath + "visualization/structures"));
        Files.createDirectories(Paths.get(rootPath + "report"));
    }
    
    public void writeJson(String filePath, Object data) {
        String json = objectMapper.writeValueAsString(data);
        Files.write(Paths.get(filePath), json.getBytes());
    }
    
    public <T> T readJson(String filePath, Class<T> clazz) {
        String json = new String(Files.readAllBytes(Paths.get(filePath)));
        return objectMapper.readValue(json, clazz);
    }
}
```
