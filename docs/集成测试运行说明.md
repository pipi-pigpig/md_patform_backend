## 1. 前置条件

### 1.1 Docker Desktop安装和启动

在运行集成测试之前，需要确保Docker环境已正确安装并启动。

**Windows系统安装步骤**：

1. 从 [Docker官网](https://www.docker.com/products/docker-desktop) 下载Docker Desktop
2. 安装完成后启动Docker Desktop
3. 等待Docker Desktop启动完成（状态栏图标显示绿色）
4. 验证Docker是否正常运行：
   ```powershell
   docker --version
   docker info
   ```

**验证成功标志**：
- `docker --version` 输出类似：`Docker version 24.x.x, build xxxxx`
- `docker info` 输出包含 `Server Version` 和 `Operating System` 信息

### 1.2 md-engine容器启动命令

md-engine容器是MD计算平台的核心计算容器，包含LAMMPS、Packmol、GROMACS等分子动力学工具。

**构建容器镜像**（首次运行）：

```powershell
# 进入项目根目录
cd d:\electrolyte-md-platform

# 使用docker-compose构建所有服务
docker-compose build md-engine

# 或单独构建md-engine镜像
docker build -t md-engine -f docker/lammps/Dockerfile docker/lammps/
```

**启动容器**：

```powershell
# 使用docker-compose启动（推荐）
docker-compose up -d md-engine

# 或单独启动容器
docker run -d --name md-engine --gpus all md-engine

# 验证容器状态
docker ps --filter "name=md-engine"
```

**验证容器运行状态**：
- 容器状态显示为 `Up`
- 容器名称为 `md-engine` 或 `md_engine`

### 1.3 文件路径挂载配置

根据项目设计，md-engine容器需要挂载以下目录：

| 容器内路径 | 宿主机路径 | 用途 |
|-----------|-----------|------|
| `/workspace/inputs` | `./data/inputs` | 输入文件目录 |
| `/workspace/results` | `./data/results` | 输出结果目录 |
| `/workspace/scripts` | `./scripts` | 脚本文件目录 |
| `/workspace/data` | `./data/md_platform_data` | 平台数据目录 |

**docker-compose.yml中的挂载配置**：

```yaml
volumes:
  - ./data/inputs:/workspace/inputs
  - ./data/results:/workspace/results
  - ./scripts:/workspace/scripts
  - ./data/md_platform_data:/workspace/data
```

**验证挂载是否生效**：

```powershell
# 进入容器检查目录
docker exec md-engine ls -la /workspace/

# 应看到以下目录
# inputs  results  scripts  data
```

## 4. 常见问题

### 4.1 Docker连接失败的解决方案

**问题表现**：
- `docker info` 命令返回错误
- `Docker命令未找到` 或 `Docker Daemon连接失败`
- Docker Desktop状态栏图标显示红色或黄色

**解决方案**：

| 问题原因 | 解决方法 |
|---------|---------|
| Docker Desktop未启动 | 启动Docker Desktop，等待绿色状态 |
| Docker服务未运行（Linux） | 执行 `sudo systemctl start docker` |
| 用户无Docker权限（Linux） | 执行 `sudo usermod -aG docker $USER`，重新登录 |
| Docker安装损坏 | 重新安装Docker Desktop |
| WSL2问题（Windows） | 在Docker Desktop设置中启用WSL2集成 |

**Windows WSL2配置**：

```powershell
# 检查WSL2是否安装
wsl --list --verbose

# 如果未安装，执行
wsl --install

# 在Docker Desktop设置中启用WSL2
# Settings -> Resources -> WSL Integration -> Enable for your distro
```

### 4.2 容器未运行的解决方案

**问题表现**：
- `md-engine容器未运行`
- `未找到运行中的md-engine容器`
- 容器状态显示 `Exited` 或 `Created`

**解决方案**：

| 容器状态 | 解决方法 |
|---------|---------|
| Exited（已停止） | `docker start md-engine` |
| Created（未启动） | `docker start md-engine` |
| 容器不存在 | 使用docker-compose构建并启动 |

**启动容器命令**：

```powershell
# 查看所有容器（包括停止的）
docker ps -a --filter "name=md-engine"

# 启动已存在的容器
docker start md-engine

# 如果容器不存在，使用docker-compose
cd d:\electrolyte-md-platform
docker-compose up -d md-engine

# 验证容器状态
docker ps --filter "name=md-engine"
```

**容器启动后验证**：

```powershell
# 检查容器日志
docker logs md-engine --tail 50

# 进入容器检查
docker exec -it md-engine bash

# 在容器内执行测试
docker exec md-engine test-md
```

### 4.3 Packmol不可用的解决方案

**问题表现**：
- `Packmol未安装或不在PATH中`
- `which packmol` 返回空
- Packmol执行失败

**解决方案**：

| 解决方式 | 操作步骤 |
|---------|---------|
| 方式1：apt安装 | `docker exec md-engine apt-get update && apt-get install -y packmol` |
| 方式2：修改Dockerfile | 在Dockerfile中添加 `RUN apt-get install -y packmol`，重新构建镜像 |
| 方式3：源码编译 | 下载源码并编译安装 |

**方式1：容器内安装（临时）**：

```powershell
# 更新并安装packmol
docker exec md-engine apt-get update
docker exec md-engine apt-get install -y packmol

# 验证安装
docker exec md-engine which packmol
docker exec md-engine packmol
```

**方式2：修改Dockerfile（永久）**：

在 `docker/lammps/Dockerfile` 中确认包含以下内容：

```dockerfile
# 安装 packmol 和其他系统工具
RUN apt-get update && apt-get install -y \
    packmol \
    gromacs \
    vmd \
    && rm -rf /var/lib/apt/lists/*
```

重新构建镜像：

```powershell
cd d:\electrolyte-md-platform
docker-compose build md-engine
docker-compose up -d md-engine
```

**方式3：源码编译**：

```powershell
# 进入容器
docker exec -it md-engine bash

# 在容器内执行
wget https://github.com/m3g/packmol/archive/refs/heads/master.zip
unzip master.zip
cd packmol-master
./configure && make
cp packmol /usr/local/bin/
```

### 5.2 失败原因分析

**常见失败原因及分析**：

| 失败现象 | 可能原因 | 分析方法 |
|---------|---------|---------|
| Docker连接失败 | Docker服务未启动 | 检查Docker Desktop状态，查看系统服务 |
| 容器未运行 | 容器被停止或崩溃 | 查看容器日志 `docker logs md-engine` |
| Packmol执行超时 | 容器资源不足或挂载问题 | 检查容器内存、CPU限制，验证目录挂载 |
| PDB文件未生成 | Packmol输入错误或权限问题 | 检查packmol.inp内容，验证文件权限 |
| 原子数量不匹配 | Packmol堆积失败或部分成功 | 查看Packmol输出日志，检查错误信息 |
| Java测试跳过 | DockerClient未注入或配置问题 | 检查application-test.yml配置 |

**日志分析方法**：

```powershell
# 查看Docker服务日志（Windows）
# 在Docker Desktop中查看日志

# 查看容器日志
docker logs md-engine --tail 100

# 查看Packmol执行日志
type temp\integration_test\output\packmol_execution.log

# 查看Java测试日志
type backend\target\surefire-reports\*.txt
```

**错误日志示例分析**：

```

Packmol execution failed
Error: Molecule overlap detected
Solution: Increase tolerance or box size

# 分析：分子堆积时发生重叠，需要调整tolerance参数或增大盒子尺寸
```

```
# Docker连接失败示例

Solution: 确认Docker Desktop是否正在运行

# 分析：Docker守护进程未运行，需要启动Docker Desktop
```
